La herramienta implementada sigue un flujo de trabajo definido que automatiza de forma secuencial las tareas más relevantes en la fase de reconocimiento de una auditoría de seguridad. A continuación, se describen detalladamente cada una de las fases.

\section{Escaneo de puertos y servicios}

La primera fase consiste en la ejecución de un escaneo con \texttt{Nmap}, una herramienta ampliamente utilizada en entornos profesionales. Nmap permite identificar los puertos abiertos en el objetivo, los servicios que están corriendo sobre esos puertos y la versión específica de dichos servicios. Esta información resulta crítica para detectar superficies expuestas, planificar el resto del análisis y orientar futuras fases como la enumeración de vulnerabilidades o fuerza bruta de credenciales.

\section{Enumeración de dominio}

En caso de que el objetivo introducido sea un dominio (en lugar de una dirección IP), se activan tres módulos adicionales que permiten obtener información detallada sobre el mismo.

\subsection{Reconocimiento DNS}

Este módulo se encarga de realizar un reconocimiento pasivo del dominio, recopilando información pública y registros DNS. Se utilizan varias herramientas para llevar a cabo esta tarea:

\begin{itemize}
  \item \texttt{dnsdumpster}: Se encarga de consultar y visualizar registros DNS del dominio, como \texttt{MX} (servidores de correo), \texttt{NS} (servidores de nombres) y \texttt{TXT} (configuraciones SPF, DKIM, etc).
  \item \texttt{mxtoolbox}: Se utiliza para consultar el registro \texttt{DMARC}, que proporciona mecanismos de protección ante ataques de suplantación de identidad vía correo electrónico.
  \item \texttt{API Ninja (WHOIS)}: Ejecuta consultas WHOIS para obtener información relacionada con la propiedad del dominio, fechas de creación y expiración, y otros datos relevantes.
\end{itemize}

\subsection{OSINT de direcciones de correo electrónico}

Tras el análisis DNS, se ejecuta un módulo de OSINT para obtener direcciones de correo electrónico asociadas al dominio, y verificar si estas han sido filtradas en alguna brecha de seguridad:

\begin{itemize}
  \item \texttt{IntelligenceX}: Busca direcciones de correo filtradas asociadas al dominio en bases de datos públicas.
  \item \texttt{LeakCheck}: Comprueba si las direcciones encontradas han sido expuestas, en qué brechas concretas, y qué información fue filtrada. Esta herramienta fue elegida tras descartar alternativas como "Have I Been Pwned" o "DeHashed", que requerían suscripción o tenían limitaciones excesivas.
\end{itemize}

\subsection{Enumeración de subdominios}

Para ampliar la superficie de ataque, se ejecuta \texttt{Subfinder}, una herramienta que utiliza múltiples fuentes para obtener subdominios asociados al dominio objetivo. Posteriormente, se lanza \texttt{Httpx} para verificar cuáles de ellos están activos, comprobando su código de respuesta HTTP.

\section{Reconocimiento web}

Si se detecta que el puerto 80 está abierto, se lanzan una serie de modulos para realizar un reconocimiento exhaustivo del servicio web.

\subsection{Escaneo de vulnerabilidades web}

Se procede a escanear el objetivo con \texttt{Nuclei}, una herramienta moderna, flexible y en constante actualización. Permite identificar vulnerabilidades comunes y debilidades en aplicaciones web. Si el usuario activa el modo de \textit{escaneo exhaustivo}, Nuclei se ejecutará también sobre todos los subdominios encontrados anteriormente.

\subsection{Detección de CMS}

Se utiliza \texttt{WhatWeb} para identificar si el sitio web analizado utiliza un gestor de contenidos (CMS) como WordPress o Joomla. Esta detección permite habilitar fases específicas posteriores, optimizadas para cada plataforma. Además, en caso de que WhatWeb no pueda identificar CMS, se revisa si alguno de los directorios enumerados empieza por \texttt{wp-} para determinar si se trata de un sitio WordPress.

\subsection{Captura de pantalla del sitio web}

Para documentar visualmente el estado del objetivo, se emplea la librería \texttt{pyppeteer}, que interactúa con el navegador "headless" de Chrome para tomar una captura del sitio web. Esta imagen se incluye en el informe final.

\subsection{Enumeración de directorios y archivos}

En esta fase se utiliza \texttt{Feroxbuster}, una herramienta moderna con alto rendimiento, recursividad automática y gran capacidad de gestión de respuestas HTTP. Permite enumerar rutas y archivos ocultos que puedan resultar de interés para el auditor. Si se activa el modo de escaneo exhaustivo, la herramienta realiza una recursividad de nivel 2, enumerando directorios dentro de otros.

\subsection{Reconocimiento de WordPress}

Si el CMS detectado es WordPress, se lanza \texttt{WPScan} para enumerar:
\begin{itemize}
  \item Usuarios del sitio, que se almacenan en una wordlist para fases posteriores.
  \item Plugins instalados, que se cruzan con bases de datos públicas para buscar CVEs. Si existe un exploit público en GitHub, se clona automáticamente.
\end{itemize}

En caso de disponer de usuarios enumerados, se lanza un ataque de fuerza bruta mediante \texttt{WPScan}, apuntando al recurso \texttt{xmlrpc.php}, que permite realizar múltiples intentos de autenticación sin restricciones. Si no se encontraron usuarios, se utiliza una wordlist por defecto con credenciales comunes.

\subsection{Reconocimiento de Joomla}

Si se detecta Joomla como CMS, se ejecuta \texttt{Joomscan}, que analiza vulnerabilidades conocidas, archivos de configuración expuestos, copias de seguridad accesibles, y posibles fallos asociados a versiones antiguas del sistema.

\section{Enumeración del servicio FTP}

Si el puerto 21 (FTP) está abierto, se utiliza la librería \texttt{ftplib} para interactuar con el servicio. Se comprueba:
\begin{itemize}
  \item Si permite login anónimo.
  \item Si existen credenciales válidas (mediante fuerza bruta).
  \item Si se obtienen permisos de lectura/escritura.
  \item Se descarga el contenido del servidor y se analiza con \texttt{Searchsploit} por si existen vulnerabilidades relacionadas con la versión del servicio.
\end{itemize}

\section{Reconocimiento del servicio SSH}

Si el puerto 22 está abierto, se lanza un ataque de fuerza bruta con \texttt{Hydra}. Inicialmente se consideró la librería \texttt{paramiko}, pero fue descartada por no estar diseñada para este propósito. Se utilizan las mismas wordlists generadas anteriormente si existen usuarios. Además, se analiza la versión del servicio con \texttt{Searchsploit} para comprobar posibles vulnerabilidades conocidas.

\section{Generación de reportes}

Una vez finalizado el análisis, la herramienta genera un informe en formato PDF con toda la información recopilada, organizada de forma clara y estructurada. Si el escaneo se ha realizado dentro de un proyecto, el informe se versiona automáticamente, permitiendo mantener un historial completo de auditorías sobre el mismo objetivo.

\section{Funcionalidades adicionales}

\begin{itemize}
  \item \textbf{Asistente de IA}: El sistema integra un modelo generativo (Gemini) que permite al usuario consultar recomendaciones, próximos pasos, análisis de resultados o dudas específicas basadas en el escaneo realizado.
  \item \textbf{Generación de siguientes pasos con IA}: A partir de los resultados obtenidos, el modelo generativo sugiere los próximos pasos a seguir, facilitando la planificación de la auditoría.
  \item \textbf{Sugerencias de soluciones con IA}: El modelo generativo ofrece recomendaciones sobre posibles soluciones a las vulnerabilidades detectadas.
  \item \textbf{Modo de escaneo exhaustivo}: Amplía la profundidad del análisis activando opciones avanzadas en varias fases. Proporciona más información, aunque incrementa el tiempo total del escaneo.
  \item \textbf{Configuración de proxy}: El usuario puede configurar manualmente un proxy desde la interfaz gráfica, útil para evadir bloqueos y especialmente integrado con herramientas como \texttt{WPScan}.
  \item \textbf{Sistema de gestión de proyectos}: Desde la interfaz, el usuario puede crear y gestionar proyectos, permitiendo mantener informes versionados, comparar resultados y documentar la evolución del objetivo.
  \item \textbf{Programación de escaneos}: La herramienta permite programar escaneos automáticos a horas específicas, facilitando la planificación y ejecución de auditorías.
\end{itemize}

Extra: si se quiere lanzar desde consola, basta con ejecutar el archivo \texttt{main.py} (python3 main.py <target> [-p] [-ex]). Se le pueden especificar los siguientes parámetros:
\begin{itemize}
  \item \texttt{-p}: Asocia el escaneo a un proyecto existente, permitiendo mantener un historial de auditorías sobre el mismo objetivo.
  \item \texttt{-ex}: Activa el modo de escaneo exhaustivo.
\end{itemize}
